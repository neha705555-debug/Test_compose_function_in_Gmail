# Incubyte Manual QA Assessment

## Overview
This repository contains the manual QA assessment created for testing the Gmail Compose functionality as part of the recruitment process for the Quality Assurance Craftsperson - Manual role at Incubyte.

The assessment focuses on validating the compose email workflow using:
- Subject: `Incubyte`
- Body: `QA test for Incubyte`

---

## Files Included

### 1. Traditional Test Cases
Covers:
- Positive scenarios
- Negative scenarios
- Field validations
- Draft handling
- Error handling
- Compose workflow validations

Includes:
- Preconditions
- Test steps
- Test data
- Expected results
- Actual execution results
- Priority and status

---

### 2. BDD / Gherkin Test Cases
BDD-style scenarios written using:
- Given
- When
- Then
- And / But

The scenarios cover:
- Successful email sending
- Validation checks
- Subject handling
- Recipient validation
- Gmail compose behavior

---

### 3. Traceability Matrix
Requirement-to-test-case mapping included to ensure:
- Requirement coverage
- Scenario traceability
- Validation completeness

---

## Testing Approach
The assessment was executed using a practical manual testing approach with focus on:
- Functional validation
- User workflow validation
- Negative testing
- Exploratory thinking
- Real Gmail compose behavior

The test cases were intentionally kept execution-oriented and concise.
